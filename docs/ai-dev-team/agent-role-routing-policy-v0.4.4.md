# エージェント・ロール・ルーティング方針 v0.4.4

タスクの内容に応じて最適な AI エージェントを選択するためのルーティングルール。

---

## ルーティング・マトリックス

| タスク内容 | 第一選択エージェント | 理由 |
|---|---|---|
| **全体設計・工程管理** | PM Agent | プロジェクトの文脈とゴールを把握しているため |
| **GCP / インフラ検討** | Gemini Agent | Google Cloud 関連の知識が豊富で最新情報に強いため |
| **ドキュメント要約・下読み** | Gemini Agent | コンテキストウィンドウが広く、大量の情報を処理できるため |
| **コード実装・バグ修正** | Claude Code | ツール実行能力が高く、修正・検証のサイクルを回せるため |
| **リファクタリング・クリーンアップ** | Claude Code | 既存コードの文脈を読み、一貫性を保った変更が得意なため |
| **最終承認・Go/No-Go 判断** | 人間 (じゅんやさん) | 責任を負い、ビジネス的な最終判断を下すため |

---

## ルーティング・フロー

1. **タスク発生**: PM Agent または人間がタスクを起票。
2. **特性分類**: タスクが「設計」「調査」「実装」「検証」のどこに属するかを分類。
3. **エージェント割当**: 上記マトリックスに基づき、最適なエージェントへ `MultiAgentTaskPacket` を発行。
4. **結果回収**: 実行エージェントからの出力を PM Agent が回収・統合。

---

## ルーティングの例外

- **特急対応**: 調査と実装を同時に行う必要がある場合は、Claude Code に直接依頼することがある。
- **高難度課題**: 一つのエージェントで解決できない場合は、複数のエージェントに異なる観点で依頼し、人間が統合する。
