# バグ修復ルーティングガイド v0.4.6

バグの性質に応じた修復担当エージェントの決定指針。

---

## ルーティング基準

| バグの性質 | 担当 | 理由 |
|---|---|---|
| **単純なバグ (Syntax, Typo)** | Claude Code | 即座に特定・修正が可能 |
| **ロジックの不整合** | Claude Code | テストを回しながらの試行錯誤が必要 |
| **仕様の解釈ミス** | PM Agent | 元のチケットと照らし合わせ、正しい仕様を再定義する必要がある |
| **インフラ起因の不具合** | Gemini Agent | GCP 設定や環境差異の分析に長けているため |
| **セキュリティ上の脆弱性** | 人間 + Gemini | 慎重な判断と多角的なレビューが必要 |

---

## 修復サイクル

1. **検知**: テスト失敗。
2. **分析 (Triage)**: `verify-failure-triage-packet.js` による自動分類。
3. **アサイン**: 上記ガイドに基づき担当を決定。
4. **修正実行**: 担当エージェントが修正パッチを作成。
5. **検証**: `npm run verify` による再確認。
6. **完了**: PM Agent による最終承認。
