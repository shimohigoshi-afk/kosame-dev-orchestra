# Claude Code 承認ポリシー v0.4.3

KOSAME Dev Orchestra における Claude Code のコマンド実行承認方針。

---

## 概要

「YES地獄（過剰な承認確認）」を削減し、安全かつ迅速な開発を実現するための承認ポリシー。
リスクレベルに応じて「自律許可」「確認必須（Ask）」「原則禁止（Deny）」を明確に区分する。

---

## 承認カテゴリ

| カテゴリ | 説明 | 具体的な対応 |
|---|---|---|
| **Allowed (自律許可)** | リスクが極めて低い操作 | ユーザーへの確認なしに実行してよい |
| **Ask (承認必須)** | 破壊的または外部影響がある操作 | 実行前に必ずユーザーの承認を得る |
| **Deny (原則禁止)** | セキュリティ上の重大なリスクがある操作 | 実行しない（または代替手段を検討する） |

---

## コマンド別ポリシー（抜粋）

| コマンド | カテゴリ | 備考 |
|---|---|---|
| `ls`, `grep`, `cat`, `find` | Allowed | 読み取り専用操作 |
| `git status`, `git diff` | Allowed | 読み取り専用操作 |
| `npm run smoke:*` | Allowed | 検証用スクリプト |
| `npm run verify` | Allowed | 全体検証スクリプト |
| `git add`, `git mv` | Allowed | ステージング操作（commit 前なら安全） |
| `git commit` | Ask | 変更の確定 |
| `git push` | Ask | リモート反映（取り消し困難） |
| `npm install` | Ask | 依存関係の変更 |
| `gcloud` / `railway` | Ask | インフラ操作（本番影響） |
| `rm -rf` | Deny | 不可逆な大規模削除 |
| `git reset --hard` | Deny | 作業履歴の破壊 |
| `env`, `printenv` | Deny | 秘密情報の漏洩リスク |

---

## 運用ルール

1. **ポリシーの事前提示**: 作業開始時に、現在の作業範囲における承認カテゴリを提示する。
2. **YES地獄の回避**: Allowed カテゴリの操作をまとめて実行（Batching）し、逐一確認を求めない。
3. **リスクの高いファイル**: `package.json` や `Dockerfile` などの重要ファイルの編集は、内容に応じて Ask に昇格させる場合がある。
