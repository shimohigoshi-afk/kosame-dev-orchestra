# Claude Code 開発ポリシー v0.4.5

Claude Code による「修復・クリーンアップ」フェーズの運用ルール。

---

## 役割: 修復とクリーンアップ

Claude Code は、その高度な推論能力とツール実行能力を活かし、実装の最終仕上げと問題解決を担当する。

### 1. 実装と修復 (Repair)
- Gemini が作成したドラフトを元にした、動作可能なコードの実装。
- テスト失敗の分析と自動修正。
- 複雑なロジックのデバッグ。

### 2. クリーンアップ (Cleanup)
- コードスタイルの統一（Linter の適用）。
- 重複コードの削除。
- 不要なコメントやデバッグログの除去。

---

## 運用ルール

- **自律検証の徹底**: ファイル編集後は、必ず関連する `smoke` テストまたは `npm run verify` を実行し、デグレードがないことを確認する。
- **承認ゲートの遵守**: `claude-code-approval-policy-v0.4.3.md` に従い、必要な場面で Ask を発行する。
- **Git 操作の規律**: 適切なコミットメッセージを作成し、ステージングまでを行う（Push は Ask カテゴリ）。

---

## 禁止事項

- **テストなしの修正**: 検証手段がない状態でのコード変更は避ける。
- **大規模リファクタリングの強行**: 事前の合意なしに、プロジェクト全体の構造を大きく変える変更を行わない。
