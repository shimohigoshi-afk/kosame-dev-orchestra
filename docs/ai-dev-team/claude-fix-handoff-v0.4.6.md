# Claude Fix Handoff v0.4.6

エラー発生時に Claude Code へ修正タスクを引き継ぐための手順。

---

## 概要

`npm run verify` や `smoke` テストで失敗が検出された際、人間の手を介さずに（または最小限の介在で）Claude Code に修正を依頼するためのフロー。

---

## ハンドオフ・フロー

1. **エラー検知**: テストランナーが非ゼロの終了コードを返却。
2. **ログ解析**: PM Agent が標準出力/標準エラー出力を解析し、失敗したテストとエラー箇所を特定。
3. **パケット生成**: `verify-failure-triage-packet.js` を使用して `ClaudeFixPacket` を生成。
4. **タスク発行**: Claude Code に対して、生成されたパケットを渡し、修正を指示。

---

## 依頼時の必須情報

Claude Code が効率的に修正を行うため、以下の情報をパケットに含める：

- **エラーメッセージ**: スタックトレースを含む生のエラーログ。
- **失敗したファイルパス**: 修正対象となる可能性が高いファイル。
- **再現コマンド**: 修正を確認するために実行すべき最小限のコマンド（例：`node smoke/target-test.js`）。

---

## 成功定義

- 該当のテストがパスすること。
- `npm run verify` 全体がパスすること。
- 新たな `smoke` テストが追加され、再発防止がなされていること（推奨）。
