# Gemini Agent 開発ポリシー v0.4.5

Gemini エージェントによる「量産・下読み」フェーズの運用ルール。

---

## 役割: 量産と下読み

Gemini は、その高いスループットと広いコンテキストウィンドウを活かし、開発の初期段階および周辺タスクを担当する。

### 1. ドキュメントの量産
- 設計メモ、テスト仕様書、README のドラフト作成。
- 既存ドキュメントのバージョンアップ（v0.x.x のインクリメント）。

### 2. コードの下読み (Pre-reading)
- 修正対象コードの構造分析。
- 依存関係の洗い出し。
- GCP ベストプラクティスとの乖離チェック。

---

## 運用ルール

- **直接編集の禁止**: Gemini が生成したコードやドキュメントは、必ず Claude Code または人間がレビューし、最終的な取り込みを行う。
- **指示の標準化**: `GeminiAgentTaskPacket` を使用し、出力形式（JSON 等）を指定する。
- **GCP 優先**: GCP 関連の調査や設定ファイルのレビューは、最優先で Gemini に割り当てる。

---

## 禁止事項

- **機密情報の投入**: API キーや環境変数の生データをプロンプトに含めない。
- **自律デプロイ**: Gemini の判断のみでインフラ構成を変更しない。
