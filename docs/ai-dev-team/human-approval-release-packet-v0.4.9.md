# Human Approval Release Packet — v0.4.9

## 概要

本番デプロイ前に人間（じゅんやさん）に提出する、承認依頼パケットの構成。

---

## パケット構成要素

承認依頼には、以下の情報が含まれていなければならない。

1. **リリースバージョン**: 今回適用する新バージョン。
2. **主な変更点**: ユーザまたは運用に与える影響の要約。
3. **検証結果**: Smoke test の実行結果（PASS/FAIL 数）。
4. **リスク評価**: デプロイに伴うリスクと、失敗時の切り戻し手順。
5. **承認用コマンド**: 承認者が実行する（または実行を許可する）コマンド。

---

## 提出テンプレート（例）

```text
【リリース承認依頼】
バージョン: v0.5.0
概要: 運用監視・コスト制御・リリースガバナンス拡張パックの統合
検証: smoke test 32件 PASS / 0件 FAIL
リスク: 既存のルーティングロジックへの影響（低）
切り戻し: gcloud run services update-traffic ...
承認をお願いします。
```
