# KOSAME PM レビューポリシー v0.4.5

PM Agent による意思決定ゲートの運用ルール。

---

## 概要

各エージェント（Gemini / Claude）の成果物を統合し、最終的な品質と進行を管理する。

---

## レビューの観点

1. **仕様適合性**: 実装がチケット（`tickets/`）や仕様書（`docs/`）の要求を満たしているか。
2. **安全性**: セキュリティリスク（APIキー漏洩等）がないか。
3. **継続性**: ドキュメントが適切に更新され、後続の作業に支障がないか。

---

## 意思決定フロー

- **自動パス (L1)**: `npm run verify` がパスし、かつドキュメントのみの変更であれば、PM Agent が承認してステージングを進める。
- **人間へのエスカレーション (L2/L3)**:
    - 破壊的変更が含まれる場合。
    - エージェント間で意見が対立した場合。
    - 予算やスケジュールに影響が出る判断。

---

## 運用ルール

- **証跡の記録**: 各エージェントからの報告をまとめ、`tickets/` 配下の完了報告を更新する。
- **フィードバックループ**: 修正が必要な場合は、適切なエージェント（例：バグなら Claude）に `TaskPacket` を再発行する。
