# Runtime Health Signal Guide — v0.4.7

## 概要

PM Agent のランタイムから発せられる「シグナル」の解釈と、それに対する対応手順を定義する。

---

## ヘルスシグナル一覧

### 1. HTTP 500 (Internal Server Error)
- **解釈**: エージェント内部の例外、またはモデル API への接続失敗。
- **対応**: `runtime-log-review-packet` を使用して、スタックトレースを確認する。

### 2. HTTP 429 (Too Many Requests)
- **解釈**: モデル API のレートリミット、または Cloud Run の同時実行数制限。
- **対応**: クォータを確認し、必要に応じてバックオフを調整またはリミット緩和を申請。

### 3. 高レイテンシ (> 30s)
- **解釈**: モデルの応答遅延、またはコールドスタートの発生。
- **対応**: リビジョン設定の `min-instances` を検討（ただしコスト増に注意）。

### 4. メモリ不足 (OOM Kill)
- **解釈**: 大規模なコンテキスト処理によるメモリ不足。
- **対応**: `max-instances` のメモリ割り当てを増強するか、入力サイズを制限する。

---

## 緊急停止判断基準

以下の場合は直ちにサービスを停止（`max-instances: 0`）し、じゅんやさんに報告する。

- 短時間に 1,000 件以上のエラーが発生。
- 課金が急激にスパイク（1時間で ¥1,000 以上など）。
- 秘密情報の漏洩が疑われるログ出力を確認。
