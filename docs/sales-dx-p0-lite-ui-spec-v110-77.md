# 営業DX P0 Lite 1画面UI仕様書

> バージョン: 110.77.0  
> 種別: UI設計パック（実装前仕様定義）  
> 前身: v110.76 CLI demo renderer  

## 目的

CLIデモ（v110.76）から1画面Web UIデモへ進めるためのUI設計を定義する。  
本ドキュメントはUI実装の仕様であり、実Web UIの実装は行わない。

## 画面構成

### 1. タイトル領域

- タイトル: **営業DX P0 Lite**
- サブタイトル: 面談メモから、議事録下書き・温度感・追客文案を整理

### 2. 入力エリア

| 項目 | 種別 | 説明 |
|------|------|------|
| 面談メモ入力 | textarea | 自由記述。面談メモを入力 |
| caseName入力 | input | ケース名（顧客名や案件名） |
| サンプルfixture選択 | select/dropdown | 事前定義済みサンプルを選択して入力欄に反映 |
| 解析ボタン | button | クリックで解析を実行 |

### 3. 出力エリア（解析実行後に表示）

| 出力項目 | 説明 |
|----------|------|
| 温度感 | high / medium_caution / medium / low / info_low の5段階とラベル表示 |
| 温度感理由 | なぜその温度感と判定したかの簡潔な説明 |
| 警戒・迷い・前向きワード | 検出されたキーワード一覧 |
| コンプラ簡易チェック | コンプラ上の注意点があれば表示 |
| 議事録下書き | 面談メモを整理した議事録案 |
| 次回対応TODO | 次回面談までに必要なTODO |
| 保留・確認事項 | 保留・確認が必要な項目 |
| 追客文下書き | 追客メール/連絡文案 |

### 4. 安全表示（常時表示）

- ⚠ この出力はAIによる下書きです
- ⚠ 最終確認は人間が行ってください
- ⚠ 簡易チェックであり、法令遵守を完全に保証するものではありません

### 5. Dry Run表示（常時表示）

| 項目 | 状態 |
|------|------|
| 保存なし | ✅ |
| 送信なし | ✅ |
| 課金なし | ✅ |
| 外部API接続なし | ✅ |

## 入出力データ仕様

### 入力

```json
{
  "text": "面談メモのテキスト",
  "caseName": "ケース名（任意）"
}
```

### 出力（解析結果）

```json
{
  "ok": true,
  "temperature": { "level": "high", "label": "高温度_前向き" },
  "temperatureReason": "肯定的な発言が多数含まれているため",
  "alertWords": {
    "positive": [{ "word": "...", "count": 1 }],
    "guard": [{ "word": "...", "count": 1 }],
    "hesitate": [{ "word": "...", "count": 1 }]
  },
  "compliance": { "warnings": [], "passed": true },
  "transcript": { "summary": "...", "pendingItems": [] },
  "nextTodos": ["..."],
  "followupDraft": "...",
  "dryRun": true,
  "humanGateRequired": true,
  "saved": false,
  "sent": false,
  "charged": false,
  "externalApiCalled": false

```

## 禁止表現カテゴリ（出力に含めてはいけない表現カテゴリ）

- AIが最終判断するような表現（例: "AIが~を判断" 等）
- 結果を断定する表現（例: "~と決定" 等）
- 精度を保証する表現
- 法令遵守を完全に保証する表現
- 最適性を謳う表現
- 全てを自動化する表現

## 安全ゲート

1. 全ての出力は `humanGateRequired: true` を設定する
2. 全ての処理は `dryRun: true` モードでのみ動作する
3. 保存・送信・課金・外部API接続は一切行わない
4. 秘密鍵・API key・環境設定値の確認・読み込みを行わない
5. 顧客情報をファイルやDBに保存しない

## 将来のUI実装時の注意事項

- 本仕様書に基づきUIを実装すること
- UIフレームワークは任意だが、現時点ではReact/Next/Vite等は導入しない
- DB保存・実API接続・実送信・実課金は行わない（Dry Runモード維持）
- 録音アップロード・リアルタイム文字起こし・音声トーン分析はスコープ外
- 文字起こし基盤本体・基幹システム基盤・営業DX本体repoは変更しない
- 人間承認前の commit/tag/push は禁止
- git add -A / force push / tag削除 / tag上書き は禁止
